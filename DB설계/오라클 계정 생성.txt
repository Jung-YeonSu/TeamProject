 명령창 CMD 실행

1) 로그인하지 않고(no login) sqlplus 실행
   C:\Users\302-00>sqlplus /nolog   


2) 관리자로 로그인
   SQL> connect /as sysdba
    연결되었습니다.
   SQL> show user
    USER은 "SYS"입니다


3) 세션 변경
oracle 11g
   계정 test
oracle 12c
   계정 c##test
oracle 12c 이상부턴 기존방식 사용할려면
  SQL> ALTER SESSION SET "_ORACLE_SCRIPT"=true;  실행 후 계정 추가


4) 계정생성
    아이디: project1  암호:1234
      SQL> create user project1 identified by 1234;
      사용자가 생성되었습니다.


 5)  권한 부여
   SQL> grant CONNECT, RESOURCE TO project1;
   권한이 부여되었습니다.
  

 6)  로그인 테스트 
   SQL>connect project1/1234
   SQL>show user

 7)  insert 시 오류발생하면 
ORA-01950: 테이블스페이스 '테이블명'에 대한 권한이 없습니다.	
원인 : 테이블 스페이스에 권한이 없다.
해석 : 계정을 생성 후, 테이블 생성 권한까지는 주었지만, 너따위한테 할당량에 대한 
      권한을 주지않았다. 고로, 할당량을 부여하는 권한이 있어야 Insert가 가능하다.

ALTER USER project1 DEFAULT TABLESPACE 
USERS QUOTA UNLIMITED ON USERS;
